# Schema And Event Contract

Agentic Factory stores durable state in a local SQLite database. The default DB
path is:

```text
.agentic-factory/factory.db
```

The schema is created from SQL migrations in `migrations/`. Runtime code enables
SQLite foreign keys with `PRAGMA foreign_keys = ON`.

## Compatibility Rules

- Treat migrations as append-only after a public release.
- Add new migrations instead of rewriting released migration files.
- Keep runtime dependencies in the Python standard library.
- Store structured extension data in `*_json` fields as JSON text.
- Use ISO 8601 UTC timestamps ending in `Z` for CLI-created records.
- Prefer additive schema changes over destructive changes.
- Document new tables, columns, event types, and payload fields here.

## Data Model Overview

The database has three layers:

- Run state: `factory_runs`, `batons`, `locks`, `actors`, `blockers`.
- Evidence records: `handoffs`, `verification_runs`, `reviews`,
  `review_findings`, `commits`.
- Append log: `events`.

The `events` table is the chronological audit trail. Current-state tables are
optimized for quick inspection and rendering.

Agent packets do not add tables. `factory.py agent packet` renders bounded
views over existing run, baton, event, verification, review, lock, and config
state.

## Tables

### `schema_migrations`

Tracks applied migration files.

| Column | Type | Notes |
| --- | --- | --- |
| `version` | INTEGER PRIMARY KEY | Numeric prefix from migration filename. |
| `name` | TEXT NOT NULL | Full migration filename. |
| `applied_at` | TEXT NOT NULL | UTC timestamp when the migration was applied. |

### `factory_runs`

One row per initialized factory run.

| Column | Type | Notes |
| --- | --- | --- |
| `id` | TEXT PRIMARY KEY | Run ID. Generated by `init` unless `--run-id` is supplied. |
| `project_root` | TEXT NOT NULL | Absolute target project root. |
| `objective` | TEXT NOT NULL DEFAULT '' | Factory objective. |
| `work_mode` | TEXT NOT NULL DEFAULT 'balanced' | Mode such as `balanced`, `safe_mvp`, `strict`, or `release`. |
| `topology` | TEXT NOT NULL DEFAULT 'executive_as_ledger' | Factory topology. |
| `status` | TEXT NOT NULL DEFAULT 'active' | Current run status, commonly `active` or `paused`. |
| `started_at` | TEXT NOT NULL | UTC timestamp. |
| `updated_at` | TEXT NOT NULL | UTC timestamp. |
| `paused_at` | TEXT | Set by `pause`. |
| `resumed_at` | TEXT | Set by `resume`. |
| `metadata_json` | TEXT NOT NULL DEFAULT '{}' | Extension data. |

### `actors`

Reserved for durable role/thread metadata.

| Column | Type | Notes |
| --- | --- | --- |
| `id` | INTEGER PRIMARY KEY AUTOINCREMENT | Actor row ID. |
| `run_id` | TEXT NOT NULL | References `factory_runs(id)`. |
| `role` | TEXT NOT NULL | Role such as Executive, Builder, Reviewer, Manager, or Watcher. |
| `name` | TEXT NOT NULL DEFAULT '' | Display name. |
| `thread_id` | TEXT NOT NULL DEFAULT '' | External thread identifier if available. |
| `model` | TEXT NOT NULL DEFAULT '' | Requested model. |
| `reasoning` | TEXT NOT NULL DEFAULT '' | Requested reasoning effort. |
| `status` | TEXT NOT NULL DEFAULT 'active' | Actor status. |
| `created_at` | TEXT NOT NULL | UTC timestamp. |
| `updated_at` | TEXT NOT NULL | UTC timestamp. |
| `metadata_json` | TEXT NOT NULL DEFAULT '{}' | Extension data. |

### `batons`

Current and historical work assignments.

| Column | Type | Notes |
| --- | --- | --- |
| `id` | TEXT PRIMARY KEY | Baton ID such as `B-001`. |
| `run_id` | TEXT NOT NULL | References `factory_runs(id)`. |
| `title` | TEXT NOT NULL | Human-readable baton title. |
| `owner` | TEXT NOT NULL DEFAULT '' | Current owner role/name. |
| `owner_thread` | TEXT NOT NULL DEFAULT '' | External owner thread identifier. |
| `status` | TEXT NOT NULL DEFAULT 'assigned' | Status such as `assigned`, `handed_off`, or `accepted`. |
| `scope` | TEXT NOT NULL DEFAULT '' | Baton scope. |
| `acceptance_tier` | TEXT NOT NULL DEFAULT 'integration' | `prototype`, `integration`, or `release`. |
| `verification_level` | TEXT NOT NULL DEFAULT 'focused' | Requested verification level. |
| `model` | TEXT NOT NULL DEFAULT '' | Requested model. |
| `reasoning` | TEXT NOT NULL DEFAULT '' | Requested reasoning effort. |
| `assigned_at` | TEXT NOT NULL | UTC timestamp. |
| `handed_off_at` | TEXT | Set by `baton handoff`. |
| `accepted_at` | TEXT | Set by `baton accept`. |
| `blocked_at` | TEXT | Reserved for blocked-state tracking. |
| `commit_sha` | TEXT NOT NULL DEFAULT '' | Accepted commit SHA if recorded. |
| `summary` | TEXT NOT NULL DEFAULT '' | Latest baton summary. |
| `metadata_json` | TEXT NOT NULL DEFAULT '{}' | Extension data; handoff metadata is stored here too. |

Active baton statuses are currently:

```text
assigned, active, in_progress, handed_off, review
```

## Built-In Event Types

The CLI writes these event types directly:

| Event type | Command | Summary | Payload |
| --- | --- | --- | --- |
| `factory.started` | `init` | Objective or mode summary. | `work_mode`, `topology`, `project_root`. |
| `baton.assigned` | `baton create` | Baton title. | `owner`, `scope`, `acceptance_tier`, `verification_level`, `lock_acquired`. |
| `verification.completed` | `verify record` | `<result>: <command>`. | `result`, `command`, `package`. |
| `baton.handed_off` | `baton handoff` | Handoff summary. | `owner`, `files_changed`, `commands_run`, `verification`. |
| `review.recorded` | `review record` | Review summary or status. | `review_id`, `findings`. |
| `baton.accepted` | `baton accept` | Acceptance summary. | `commit`, `pushed_status`. |
| `factory.paused` | `pause` | Pause reason or mode. | `mode`, `reason`. |
| `factory.resumed` | `resume` | Resume reason. | `reason`. |
| `lock.acquired` | `lock acquire` | Lock holder summary. | `lock`. |
| `lock.released` | `lock release` | Lock release summary. | `lock`. |

`event append` can write custom event types. Custom event names should be
namespaced with dot notation, for example:

```text
factory.note
release.gate.started
external.blocker.recorded
```

Custom payloads must be JSON objects.

## Status And Rendering Queries

The current CLI reads:

- latest run by `factory_runs.started_at DESC`;
- active batons by `run_id` and status in the active status set;
- held locks by `run_id` and `status = 'held'`;
- latest baton by `assigned_at DESC`;
- latest event by `occurred_at DESC, id DESC`;
- rendered ledger data from recent batons and events for the current run.

When adding new inspection commands, prefer indexed access paths and bounded
queries. Avoid commands that require loading the entire event log by default.

Read-only inspection commands use the same tables:

- `baton list`: bounded query on `batons` by current `run_id` and optional
  status filters.
- `baton show`: one baton plus related handoffs, verification, reviews,
  findings, commits, and recent events.
- `events list`: bounded query on `events` by current `run_id`, optional baton,
  and optional event type.
- `verification list`: bounded query on `verification_runs`, scoped by baton or
  current run.
- `review list`: bounded query on `reviews` and `review_findings`, scoped by
  baton or current run.

Project config is file-based, not stored in SQLite. See
`docs/configuration.md` for `.agentic-factory/config.json`.

## Migration Checklist

Before adding a migration:

1. Decide whether the change is additive and backward compatible.
2. Add a new `migrations/<version>_<name>.sql` file with the next numeric prefix.
3. Update this document with new tables, columns, indexes, and event types.
4. Add or update tests that initialize a fresh DB and exercise the new behavior.
5. Run `bash scripts/check.sh`.

Avoid:

- rewriting released migration files;
- removing or renaming columns without a compatibility plan;
- changing event payload meanings without documenting migration semantics;
- adding dependencies on non-stdlib Python packages for migration execution.
